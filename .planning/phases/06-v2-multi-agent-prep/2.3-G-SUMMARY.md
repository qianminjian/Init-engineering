## 实施变更

### 1. RoundResult 新增 history 字段 (round.py)

```python
@dataclass
class RoundResult:
    round_id: int
    outcomes: list[TaskOutcome] = field(default_factory=list)
    gate_results: dict[str, Verdict] = field(default_factory=dict)
    history: list[RoundHistory] = field(default_factory=list)  # 新增
    started_at: float = 0.0
    finished_at: float = 0.0
```

### 2. run_round 末尾构造 RoundHistory

新增 `_attach_round_history()` 辅助函数, 在 run_round 末尾调用:
- 从 RoundResult 读 gate_results
- 从 outcomes 提取 task_outcomes
- 从 completed_count 算 files_changed
- 从 git diff --numstat HEAD~1 HEAD 算 lines_added/removed
- 写入 result.history (1 个元素), semantic_satisfied=None (Orchestrator 写回)

### 3. Orchestrator 简化

**删除:**
- `_build_history` 方法 (重复构造消除)
- `_parse_git_numstat` (迁入 round.py)

**新增:**
- 主循环逻辑改为: run_round → `_evaluate_semantic(round_result)`
  → 写回 `round_result.history[0].semantic_satisfied`
  → `self.history.extend(round_result.history)` (累加而非 append)
  → 收敛判定

**保留向后兼容:**
- `_run_gates` / `_evaluate_semantic` 方法保留
- `run_round` 公开 API 不变 (无新参数)

### 4. _parse_git_numstat 迁入 round.py

从 orchestrator.py 提取, 改名为模块级私有函数 `_parse_git_numstat`,
供 `_attach_round_history` 调用。git 不可用 / 仓库无 HEAD 时返回 (0, 0)。

## 借鉴 LangGraph

LangGraph `Pregel.tick()` 构造 `Packet` 直接传给 channel, 不在调用方
(PregelLoop) 重复打包数据。本 Phase G:

| LangGraph | 本项目 (v2.3-G) |
|-----------|----------------|
| `Pregel.tick()` 构造 Packet | `run_round()` 末尾构造 RoundHistory |
| channel 直接接收 Packet | `round_result.history` 直接是 RoundHistory 列表 |
| PregelLoop 不二次包装 | Orchestrator 不再 `_build_history` |

数据流:

```
旧:  run_round → round_result → _build_history(round_id, rr, tasks) → RoundHistory
新:  run_round → round_result (含 history[0]: RoundHistory) → extend orch.history
```

## 偏差记录

### 1. [Rule 2 - 必要功能] run_round 不接 semantic_satisfied 参数

**原计划:** run_round 末尾接 `semantic_satisfied: bool | None = None` 参数,
Orchestrator 注入。

**实际:** semantic_satisfied 在 Orchestrator.run() 调 `_evaluate_semantic`
后写回 `round_result.history[0].semantic_satisfied`。

**原因:** 原计划方案让 run_round 接受 Orchestrator 特有的概念, 破坏
模块边界 (round.py 不应依赖 semantic_evaluator)。实际方案保持 run_round
public API 不变, semantic_satisfied 由 Orchestrator 写回, 更清晰。

**影响:** 0 回归。3 新增测试 + 113 既有测试全通过。

### 2. [Rule 2 - 必要功能] 保留 _run_gates 方法

**计划:** Orchestrator 删除 _build_history + _parse_git_numstat。

**实际:** `_run_gates` 方法保留 (向后兼容, 标记 deprecated), `_evaluate_semantic`
保留 (用于回写)。`_parse_git_numstat` 迁入 round.py 模块级私有函数。

**原因:** 外部代码可能 import `_run_gates` (虽是私有约定但实际有调用),
保持向后兼容, 加 docstring 标记为 legacy。

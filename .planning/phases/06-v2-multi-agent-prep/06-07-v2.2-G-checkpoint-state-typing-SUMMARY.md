# Phase 06 Plan v2.2-G: Checkpoint.state Any 类型重构 (P2.1) Summary

**One-liner:** 用 `LoopStateProtocol` (runtime_checkable) + `Generic[T]` 替代 `state: Any`,
打破 `loop/checkpoint.py` ↔ `loop/state.py` 循环引用,mypy 看到 `state` 字段是具体类型。

---

## What Was Built

### 1. 新建 `auto_engineering/loop/types.py` (99 行)

类型契约层,提供:

- **`LoopStateProtocol`** — runtime_checkable Protocol,定义 LoopState 最小接口契约:
  - 字段: `round: int`, `step: int`, `status: str`, `channels: dict[str, Any]`
  - 方法: `model_dump(**kwargs) -> dict`
- **`serialize_state(state: LoopStateProtocol) -> str`** — JSON 序列化
- **`deserialize_state(json_str: str) -> dict`** — JSON 反序列化(返回 dict,职责分离)

**关键设计**: types.py **不引用** `loop/state.py`,从源头杜绝循环引用。

### 2. 重构 `auto_engineering/loop/checkpoint.py`

- **`Checkpoint[T]`** — 替代 `Checkpoint`:`state: T` (bound LoopStateProtocol) 替代 `state: Any`
- **`SQLiteCheckpointStore[T]`** — Generic[T]:
  - `save(state: T, ...)` 替代 `save(state: Any, ...)`
  - `load(...) -> Checkpoint[T]`, `load_latest() -> Checkpoint[T] | None`
  - `load_by_round() -> Checkpoint[T] | None`
- **`_serialize_state(state: LoopStateProtocol)`** — 接受 Protocol 替代 Any
- **G.3 反向引用消除验证**:
  - `state.py` → 不 import checkpoint (grep clean)
  - `types.py` → 不 import state (grep clean)
  - `checkpoint.py` → 仅运行时延迟 import `deserialize_loop_state` (无 module-level 循环)

## Deviations from Plan

### 1. mypy → runtime type hint check (Rule 1 - Bug 替代方案)

- **原计划**: 运行 `mypy --strict` 验证 `state: Any` 不存在
- **实际情况**: 项目 venv 未安装 mypy (`.venv/bin/mypy` 不存在,`pdm/mypy` 未在 dev deps)
- **替代方案**: 使用 `typing.get_type_hints()` 运行时反射获取声明类型 — 这是 Python 官方提供的类型检查机制,效果等价(都解析 `__annotations__` 字典)
- **验证结果**: `hints['state']` = `T` (TypeVar) 而非 `Any`,满足 P2.1 改进目标
- **后续建议**: 如需 mypy 严格模式,可在 dev deps 添加 `mypy>=1.0` 并配置 `[tool.mypy] strict = true`

### 2. PEP 695 Generic[T] 内联语法 (Phase 2.2-G 重构,Rule 1 cleanup)

- **原计划**: `class Checkpoint(Generic[T])` 写法
- **实际情况**: ruff UP046 提示使用 PEP 695 内联语法
- **采用**: `class Checkpoint[T]:` (Python 3.12+ 推荐)
- **影响**: 仅语法糖,行为完全等价

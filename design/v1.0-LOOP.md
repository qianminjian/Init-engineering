## 一、三层 Loop 架构

```
┌──────────────────────────────────────────────────────┐
│ Layer 1: LoopEngine                                  │
│   while True: tick() → execute() → after_tick()     │
│   参考 LangGraph PregelLoop (简化，去掉 superstep)     │
│   职责：控制循环、checkpoint、中断恢复                   │
├──────────────────────────────────────────────────────┤
│ Layer 2: StageGraph                                  │
│   定义 Stage 节点 + 边（固定边 + 条件边）                │
│   参考 LangGraph StateGraph (简化，去掉 channel 类型)    │
│   职责：决定"下一步是什么"、PUSH/PULL 调度               │
├──────────────────────────────────────────────────────┤
│ Layer 3: AgentRuntime                                │
│   Agent 注册 + 调用 + 中间件链                         │
│   参考 AutoGen SingleThreadedAgentRuntime (简化)       │
│   职责：Agent 执行、工具分发、Guardrail 中间件              │
└──────────────────────────────────────────────────────┘
```

### 1.1 设计原则

- **控制流在 Python**：while True 循环、Guardrail 检查、条件路由全部是 Python 代码
- **智能在 LLM**：每个 Stage 内部的决策（设计/编码/审查）由 Claude API 完成
- **Guardrail 中间件化**：Guardrail 不是硬编码在 loop 中，而是可插拔的 GuardrailHandler 链（参考 CrewAI GuardrailResult + AutoGen InterventionHandler）
- **Channel 通信**：Stage 之间通过 state channel 传递数据，不直接耦合
- **PUSH 预留**：架构层支持 Send 动态路由，为多 Agent 并行做准备

### 2.1 核心执行流程

```python
# engine/loop.py
import asyncio

class LoopEngine:
    """简化版 PregelLoop。去掉 superstep、channel 类型系统、durability 多层。
    完整实现（含重试+取消）见 Section 七.4。"""

    def __init__(
        self,
        graph: StageGraph,
        runtime: "AgentRuntime | None" = None,
        checkpoint_dir: str = ".ae-checkpoints",
        max_steps: int = 10,
        interrupt_before: set[str] | None = None,
        interrupt_after: set[str] | None = None,
    ):
        self.graph = graph
        self.runtime = runtime  # Phase 1 可为 None，传入 MockRuntime
        self.checkpoint_dir = checkpoint_dir
        self.max_steps = max_steps
        self.interrupt_before = interrupt_before or set()
        self.interrupt_after = interrupt_after or set()

        # 运行时状态（每次 run() 时重置）
        self.checkpoint: Checkpoint | None = None
        self.store: CheckpointStore | None = None
        self.current_task: Stage | None = None
        self.step: int = 0
        self.status: str = "pending"
        self._cancellation: CancellationToken | None = None

    def _init_checkpoint(self, requirement: str) -> None:
        """创建新 checkpoint 并初始化存储（仅新 run() 调用，resume() 跳过）。"""
        thread_id = str(uuid.uuid4())
        state = LoopState(requirement=requirement)
        self.checkpoint = Checkpoint.create(thread_id=thread_id, state=state)
        self.store = CheckpointStore(
            Path(self.checkpoint_dir) / f"{thread_id}.db")
        self.store.save_checkpoint(self.checkpoint)
        self.step = 0
        self.status = "pending"

    async def run(self, requirement: str = "", max_steps: int | None = None) -> LoopResult:
        """全链路 async：tick → execute → after_tick。
        若 checkpoint 已设置（来自 resume()），跳过 _init_checkpoint()。
        max_steps 默认使用构造器值，避免参数与 self.max_steps 混淆。
        """
        steps = max_steps if max_steps is not None else self.max_steps
        if self.runtime is None:
            raise AEError(ErrorCode.AGENT_REGISTRATION_ERROR,
                          "LoopEngine.runtime is None. Pass a runtime (or MockRuntime) to run().")

        if self.checkpoint is None:
            self._init_checkpoint(requirement)
        self._cancellation = CancellationToken()

        try:
            retry_count = 0
            while True:
                self._cancellation.check()
                if not self.tick(steps):
                    break
                try:
                    result = await self.runtime.execute(
                        self.current_task, self.checkpoint.state,
                        cancellation=self._cancellation,
                    )
                except OutputDropped:
                    # 静默丢弃当前 Stage 输出，跳过 after_tick，进入下一轮
                    continue
                except GuardrailRetrySignal as e:
                    retry_count += 1
                    if retry_count >= 3:
                        raise AEError(ErrorCode.STAGE_RETRY_EXCEEDED,
                                      f"Stage {self.current_task.name} 重试超过 3 次",
                                      original_error=e)
                    await asyncio.sleep(2 ** retry_count)
                    continue  # 跳过 after_tick，重试当前 Stage
                self.after_tick(result)
                retry_count = 0  # 成功后重置

                if self.status == "done":
                    break

            return LoopResult.from_checkpoint(self.checkpoint)
        except AEError as e:
            if e.code == ErrorCode.TASK_CANCELLED:
                self.checkpoint.status = "drained"
                self.store.save_checkpoint(self.checkpoint)
            raise

    def tick(self, max_steps: int) -> bool:
        """参考 LangGraph _loop.py:592-674"""
        if self.step >= max_steps:
            self.status = "out_of_steps"
            return False

        next_stage = self.graph.next_stage(self.checkpoint.state)
        if next_stage is None:
            self.status = "done"
            return False

        self.current_task = next_stage
        self.checkpoint.state.current_stage = next_stage.name  # 关键：同步到 state

        if self.interrupt_before and self.current_task.name in self.interrupt_before:
            self.status = "interrupt_before"
            return False

        return True

    def after_tick(self, result: StageResult) -> None:
        """参考 LangGraph _loop.py:676-712"""
        self.checkpoint.apply_writes(result.writes)
        self.checkpoint.increment_step()
        self.store.save_checkpoint(self.checkpoint)
        self.store.save_writes(self.checkpoint.id, self.current_task.name, result.writes)
        self.step += 1

        if self.interrupt_after and self.current_task.name in self.interrupt_after:
            self.status = "interrupt_after"

    async def resume(self, checkpoint_id: str) -> LoopResult:
        """从 checkpoint 恢复。加载 checkpoint → 设置到 self.checkpoint → 调用 run()。
        run() 检测到 self.checkpoint 已设置，跳过 _init_checkpoint()。
        """
        if self.store is None:
            raise AEError(ErrorCode.CHECKPOINT_LOAD_FAILED,
                          "Cannot resume: no checkpoint store. Run the loop first.")
        self.checkpoint = self.store.load_checkpoint(checkpoint_id)
        self.step = self.checkpoint.step + 1
        self.status = "pending"
        # checkpoint 已设置，run() 不会覆盖
        return await self.run()
```

```python
        """读取指定 channel 的值，传给 Agent 作为上下文"""
        return {n: getattr(self, n) for n in names if hasattr(self, n)}

    def set_channels(self, writes: dict) -> None:
        """写入 channel 值"""
        for k, v in writes.items():
            if hasattr(self, k):
                setattr(self, k, v)
```

### 2.3 Checkpoint Schema

```python
# engine/checkpoint.py
import json
import sqlite3
import uuid
from dataclasses import dataclass
from pathlib import Path

CREATE_CHECKPOINTS_TABLE = """
CREATE TABLE IF NOT EXISTS checkpoints (
    id              TEXT PRIMARY KEY,
    parent_id       TEXT,
    thread_id       TEXT NOT NULL,
    step            INTEGER NOT NULL DEFAULT 0,
    state_json      TEXT NOT NULL,
    status          TEXT NOT NULL DEFAULT 'pending',
    created_at      TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at      TEXT NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (parent_id) REFERENCES checkpoints(id)
);
"""

CREATE_WRITES_TABLE = """
CREATE TABLE IF NOT EXISTS writes (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    checkpoint_id   TEXT NOT NULL,
    task_id         TEXT NOT NULL,
    channel         TEXT NOT NULL,
    value_json      TEXT NOT NULL,
    FOREIGN KEY (checkpoint_id) REFERENCES checkpoints(id)
);
"""


class CheckpointStore:
    """SQLite 连接管理 + CRUD。每个 thread 一个 .db 文件，WAL 模式。"""

    def __init__(self, db_path: str | Path):
        self.db_path = Path(db_path)
        self.db_path.parent.mkdir(parents=True, exist_ok=True)
        self._conn: sqlite3.Connection | None = None

    def _get_conn(self) -> sqlite3.Connection:
        if self._conn is None:
            self._conn = sqlite3.connect(str(self.db_path))
            self._conn.execute("PRAGMA journal_mode=WAL;")
            self._conn.execute("PRAGMA foreign_keys=ON;")
            self._conn.execute(CREATE_CHECKPOINTS_TABLE)
            self._conn.execute(CREATE_WRITES_TABLE)
        return self._conn

    def save_checkpoint(self, cp: "Checkpoint") -> None:
        conn = self._get_conn()
        state_json = json.dumps(cp.state.to_dict(), ensure_ascii=False)
        conn.execute(
            """INSERT OR REPLACE INTO checkpoints (id, parent_id, thread_id, step, state_json, status, updated_at)
               VALUES (?, ?, ?, ?, ?, ?, datetime('now'))""",
            (cp.id, cp.parent_id, cp.thread_id, cp.step, state_json, cp.status),
        )
        conn.commit()

    def load_checkpoint(self, checkpoint_id: str) -> "Checkpoint":
        conn = self._get_conn()
        row = conn.execute(
            "SELECT id, parent_id, thread_id, step, state_json, status FROM checkpoints WHERE id = ?",
            (checkpoint_id,),
        ).fetchone()
        if row is None:
            raise AEError(ErrorCode.CHECKPOINT_LOAD_FAILED,
                          f"Checkpoint not found: {checkpoint_id}")
        return Checkpoint(
            id=row[0], parent_id=row[1], thread_id=row[2],
            step=row[3], state=LoopState.from_dict(json.loads(row[4])),
            status=row[5],
        )

    def save_writes(self, checkpoint_id: str, task_id: str, writes: dict) -> None:
        conn = self._get_conn()
        for channel, value in writes.items():
            conn.execute(
                "INSERT INTO writes (checkpoint_id, task_id, channel, value_json) VALUES (?, ?, ?, ?)",
                (checkpoint_id, task_id, channel, json.dumps(value, ensure_ascii=False)),
            )
        conn.commit()

    def close(self) -> None:
        if self._conn:
            self._conn.close()
            self._conn = None


@dataclass
class Checkpoint:
    id: str
    parent_id: str | None
    thread_id: str
    step: int
    state: "LoopState"
    status: str = "pending"

    @classmethod
    def create(cls, thread_id: str, state: "LoopState",
               parent_id: str | None = None) -> "Checkpoint":
        return cls(
            id=str(uuid.uuid4()),
            parent_id=parent_id,
            thread_id=thread_id,
            step=0,
            state=state,
            status="pending",
        )

    def increment_step(self) -> None:
        self.step += 1

    def apply_writes(self, writes: dict) -> None:
        self.state.set_channels(writes)

    def to_row(self) -> dict:
        return {
            "id": self.id, "parent_id": self.parent_id,
            "thread_id": self.thread_id, "step": self.step,
            "state_json": json.dumps(self.state.to_dict(), ensure_ascii=False),
            "status": self.status,
        }
```

> **设计说明**：
> - **连接管理**：`CheckpointStore` 封装 SQLite 连接，WAL 模式支持并发读、单写者。每个 `thread_id` 一个 `.db` 文件（路径：`.ae-checkpoints/{thread_id}.db`）。
> - **事务**：`save_checkpoint()` 和 `save_writes()` 各自独立事务，`commit()` 确保原子写入。
> - **序列化**：`LoopState.to_dict()` → `json.dumps()` → `state_json` TEXT 列。P0-#2（LoopState 不可 JSON 序列化）已在 state.py 中通过 `to_dict`/`from_dict` 解决。
> - **并发安全**：WAL 模式 + SQLite 文件锁。v1.0 单 loop 实例无并发写冲突。Phase 2 加 pidfile 防多实例。

```python
# engine/graph.py

@dataclass
class Stage:
    """参考 LangGraph PregelNode + CrewAI Task 富模型"""
    name: str                          # "architect" | "developer" | "critic"
    agent_type: str
    description_template: str          # 使用 {var} Python 格式化，非 Jinja2
    expected_output: str               # CrewAI 风格：Agent 知道"产出应该长什么样"
    output_schema: dict | None = None  # JSON Schema 约束 LLM 输出结构
    tools: list[str] = field(default_factory=list)
    input_channels: list[str] = field(default_factory=list)
    output_channels: list[str] = field(default_factory=list)
    retry_policy: RetryPolicy | None = None  # 每个 Stage 可独立配置重试策略

    def render_description(self, state: LoopState) -> str:
        """用 state 的 channel 值渲染描述模板。
        使用逐 key 替换而非 str.format()——避免 LLM 输出中的 { } 被误解析。
        """
        result = self.description_template
        channels = state.get_channels(self.input_channels)
        for key, value in channels.items():
            result = result.replace(f"{{{key}}}", str(value))
        return result


@dataclass
class ConditionSpec:
    """参考 LangGraph BranchSpec（简化）"""
    condition: Callable[[LoopState], str]   # 返回路径 key
    path_map: dict[str, str]               # key → Stage name 或 "END"
```

### 3.2 图构建

```python
class StageGraph:
    """参考 LangGraph StateGraph（简化：去掉 channel 类型、reducer、managed value）"""

    # LangGraph 风格 START sentinel
    START = "__start__"
    END = "__end__"

    def __init__(self):
        self.stages: dict[str, Stage] = {}
        self.edges: dict[str, str] = {}                        # 固定边
        self.conditional_edges: dict[str, ConditionSpec] = {}  # 条件边
        self._start_stage: str | None = None                   # 入口 Stage

    def set_start(self, stage_name: str) -> "StageGraph":
        """设置入口 Stage（替代 LangGraph 的 START → first_node 边）。"""
        if stage_name not in self.stages:
            raise ValueError(f"Start stage '{stage_name}' not registered")
        self._start_stage = stage_name
        return self

    def add_stage(self, stage: Stage) -> "StageGraph":
        self.stages[stage.name] = stage
        return self

    def add_edge(self, from_stage: str, to_stage: str) -> "StageGraph":
        self.edges[from_stage] = to_stage
        return self

    def add_conditional_edge(self, from_stage: str, condition: Callable,
                             path_map: dict) -> "StageGraph":
        self.conditional_edges[from_stage] = ConditionSpec(condition, path_map)
        return self

    def next_stage(self, state: LoopState) -> Stage | None:
        """参考 LangGraph prepare_next_tasks()（简化：单任务而非任务集）"""
        current = state.current_stage

        # 首 Stage：current_stage 为空时返回入口
        if not current:
            if self._start_stage:
                return self.stages[self._start_stage]
            raise AEError(ErrorCode.GRAPH_RECURSION_LIMIT,
                          "No start stage set. Call graph.set_start('stage_name').")

        # 先查条件边（如 Critic → APPROVE or MAJOR）
        if cond := self.conditional_edges.get(current):
            decision = cond.condition(state)
            next_name = cond.path_map.get(decision)
            if next_name is None or next_name == self.END:
                return None
            if next_name not in self.stages:
                raise ValueError(f"条件边指向未知 Stage: {next_name}")
            return self.stages[next_name]

        # 再查固定边
        if next_name := self.edges.get(current):
            if next_name == self.END:
                return None
            return self.stages[next_name]

        return None  # 终点
```

## 四、Layer 3: AgentRuntime

### 4.1 运行时

```python
# runtime/runtime.py

class AgentRuntime:
    """参考 AutoGen SingleThreadedAgentRuntime
    简化：去掉 pub/sub、TopicId、SerializationRegistry。保留 Send/Response + GuardrailHandler。
    """

    def __init__(self, guardrail_chain: "GuardrailChain | None" = None):
        self._agent_factories: dict[str, Callable[[], BaseAgent]] = {}
        self._agents: dict[str, BaseAgent] = {}
        self._guardrail_chain = guardrail_chain or GuardrailChain()

    def register(self, agent_type: str, factory: Callable[[], BaseAgent],
                 expected_class: type[BaseAgent] | None = None) -> None:
        """参考 AutoGen _agent_factories 延迟实例化模式 + 类型验证"""
        if expected_class is not None:
            instance = factory()
            if not isinstance(instance, expected_class):
                raise TypeError(
                    f"Agent factory for '{agent_type}' returned {type(instance).__name__}, "
                    f"expected {expected_class.__name__}"
                )
            del instance  # 即时释放测试实例
        self._agent_factories[agent_type] = factory

    def _get_or_create_agent(self, agent_type: str) -> BaseAgent:
        if agent_type not in self._agents:
            self._agents[agent_type] = self._agent_factories[agent_type]()
        return self._agents[agent_type]

    async def execute(self, stage: Stage, state: LoopState,
                      cancellation: CancellationToken | None = None) -> StageResult:
        agent = self._get_or_create_agent(stage.agent_type)

        task = Task(
            description=stage.render_description(state),
            expected_output=stage.expected_output,
            context=state.get_channels(stage.input_channels),
            tools=stage.tools,
            output_schema=stage.output_schema,
        )

        ctx = TaskContext(stage=stage.name, state=state)

        # Pre-Guardrail
        gr = await self._guardrail_chain.on_execute_start(ctx, task)
        if not gr.success:
            if gr.action == "drop":
                raise OutputDropped(gr.error)
            raise GuardrailBlockedError(gr.error)

        # Agent 执行（携带取消令牌）
        result = await agent.execute(task, ctx, cancellation=cancellation)

        # Post-Guardrail
        gr = await self._guardrail_chain.on_execute_end(ctx, result)
        if not gr.success:
            if gr.action == "retry":
                raise GuardrailRetrySignal(gr.error)

        writes = dict(zip(stage.output_channels, result.values()))
        return StageResult(stage=stage.name, writes=writes, raw=result)
```

## 五、Guardrail 系统

### 5.1 设计

**旧设计**：Gate 是二进制 `ctx.blocked = True/False`。语义单一，无法表达"丢弃消息"vs"中止循环"vs"需要重试"。

**新设计**：Guardrail = 结构化校验，参考 CrewAI `GuardrailResult` + AutoGen `InterventionHandler`。每个 Guardrail 返回 `GuardrailResult(success, result, error, action)` 而非修改 ctx。

### 5.2 GuardrailResult 模型

```python

    result: Any | None = None      # 通过时可能转换的输出
    error: str = ""                # 失败原因
    action: str = "block"          # "pass" | "block" | "drop" | "retry"
```

### 5.3 GuardrailHandler 协议

```python
class GuardrailHandler(Protocol):
    """参考 AutoGen InterventionHandler"""
    async def on_execute_start(self, ctx: TaskContext, task: Task) -> GuardrailResult: ...
    async def on_execute_end(self, ctx: TaskContext, result: TaskResult) -> GuardrailResult: ...


class DefaultGuardrailHandler:
    """默认透传：返回 success=True"""
    async def on_execute_start(self, ctx, task):
        return GuardrailResult(success=True, action="pass")
    async def on_execute_end(self, ctx, result):
        return GuardrailResult(success=True, action="pass")


class GuardrailChain:
    """中间件链。每个 Handler 返回 GuardrailResult，链根据 action 决定下一步。"""
    def __init__(self, handlers: list[GuardrailHandler] | None = None):
        self.handlers = handlers or []

    async def on_execute_start(self, ctx, task):
        for h in self.handlers:
            result = await h.on_execute_start(ctx, task)
            if not result.success:
                return result  # 短路上报
        return GuardrailResult(success=True, action="pass")

    async def on_execute_end(self, ctx, result):
        for h in self.handlers:
            gr = await h.on_execute_end(ctx, result)
            if not gr.success:
                return gr
        return GuardrailResult(success=True, action="pass")
```

### 5.6 Runtime 中的 Guardrail 集成

```python
# runtime/runtime.py — execute() 方法的 Guardrail 部分
async def execute(self, stage: Stage, state: LoopState,
                  cancellation: CancellationToken | None = None) -> StageResult:
    agent = self._get_or_create_agent(stage.agent_type)

    task = Task(
        description=stage.render_description(state),
        expected_output=stage.expected_output,
        context=state.get_channels(stage.input_channels),
        tools=stage.tools,
        output_schema=stage.output_schema,
    )

    ctx = TaskContext(stage=stage.name, state=state)

    # Pre-Guardrail
    gr = await self._guardrail_chain.on_execute_start(ctx, task)
    if not gr.success:
        if gr.action == "drop":
            raise OutputDropped(gr.error)
        raise GuardrailBlockedError(gr.error)

    # Agent 执行（携带取消令牌）
    result = await agent.execute(task, ctx, cancellation=cancellation)

    # Post-Guardrail：处理全部 4 种 action
    gr = await self._guardrail_chain.on_execute_end(ctx, result)
    if not gr.success:
        if gr.action == "retry":
            raise GuardrailRetrySignal(gr.error)
        elif gr.action == "drop":
            raise OutputDropped(gr.error)
        else:  # "block" 或未知 → 中止
            raise GuardrailBlockedError(gr.error)

    # 长度校验：避免静默丢失数据
    if len(result.values()) != len(stage.output_channels):
        raise AEError(ErrorCode.INVALID_AGENT_OUTPUT,
            f"Stage '{stage.name}': expected {len(stage.output_channels)} outputs, "
            f"got {len(result.values())}")

    writes = dict(zip(stage.output_channels, result.values()))
    return StageResult(stage=stage.name, writes=writes, raw=result)
```

### 7.2 RetryPolicy + CancellationToken

```python
# engine/retry.py
from dataclasses import dataclass, field

@dataclass
class RetryPolicy:
    """参考 LangGraph RetryPolicy"""
    max_attempts: int = 3
    initial_delay: float = 1.0
    backoff_factor: float = 2.0
    max_delay: float = 60.0
    retry_on: tuple[type[Exception], ...] = (Exception,)
    timeout: float | None = None

# engine/cancellation.py
class CancellationToken:
    """参考 AutoGen CancellationToken"""
    def __init__(self):
        self._cancelled = False

    def cancel(self) -> None:
        self._cancelled = True

    @property
    def is_cancelled(self) -> bool:
        return self._cancelled

    def check(self) -> None:
        if self._cancelled:
            raise AEError(ErrorCode.TASK_CANCELLED, "Task was cancelled")


async def run_with_retry(fn, policy: RetryPolicy,
                         cancellation: CancellationToken | None = None) -> Any:
    """参考 LangGraph run_with_retry"""
    for attempt in range(policy.max_attempts):
        if cancellation:
            cancellation.check()
        try:
            if policy.timeout:
                return await asyncio.wait_for(fn(), timeout=policy.timeout)
            return await fn()
        except policy.retry_on as e:
            if attempt == policy.max_attempts - 1:
                raise AEError(ErrorCode.LLM_MAX_RETRIES,
                              f"Failed after {policy.max_attempts} attempts",
                              original_error=e) from e
            delay = min(policy.initial_delay * (policy.backoff_factor ** attempt),
                        policy.max_delay)
            await asyncio.sleep(delay)
```

### 7.3 BaseAgent（融合优化点）

```python
# agents/base.py
from dataclasses import dataclass, field

@dataclass
class BaseAgent:
    id: str = ""
    role: str = ""
    system_prompt: str = ""
    llm: AnthropicProvider | None = None
    tools: list[BaseTool] = field(default_factory=list)
    retry_policy: RetryPolicy = field(default_factory=RetryPolicy)
    max_tool_calls: int = 10  # 防止 LLM 无限调用工具

    async def execute(self, task: Task, ctx: TaskContext,
                      cancellation: CancellationToken | None = None) -> TaskResult:
        """参考 AutoGen agent.on_message() 模式"""
        messages = self._build_messages(task, ctx)
        tool_call_log = []
        tool_call_count = 0

        try:
            response = await run_with_retry(
                lambda: self.llm.create_message(
                    model="claude-sonnet-4-6",
                    system=self.system_prompt,
                    messages=messages,
                    tools=[t.to_schema() for t in self.tools],
                    max_tokens=4096,
                ),
                self.retry_policy, cancellation,
            )

            # 工具调用循环（含上限保护）
            while response.stop_reason == "tool_use":
                if tool_call_count >= self.max_tool_calls:
                    raise AEError(ErrorCode.LLM_MAX_RETRIES,
                                  f"Exceeded max tool calls ({self.max_tool_calls})")
                for tc in response.tool_calls:
                    tool_result = self._execute_tool(tc)
                    tool_call_log.append({
                        "name": tc.name, "input": tc.input, "ok": tool_result.success
                    })
                    messages.append(self._tool_result_message(tc.id, tool_result))
                    tool_call_count += 1
                response = await run_with_retry(
                    lambda: self.llm.create_message(
                        model="claude-sonnet-4-6",
                        system=self.system_prompt,
                        messages=messages,
                        tools=[t.to_schema() for t in self.tools],
                        max_tokens=4096,
                    ),
                    self.retry_policy, cancellation,
                )

            # 解析输出（双层防御：schema 优先，fallback 正则）
            parsed = parse_agent_output(response.text, task.output_schema)
            return TaskResult(
                agent_type=self.role,
                values=self._values_from_parsed(parsed, task),
                raw_response=response.text,
                parsed_json=parsed,
                tool_calls=tool_call_log,
            )
        except AEError:
            raise
        except Exception as e:
            raise AEError(ErrorCode.LLM_TIMEOUT,
                          f"Agent {self.role} LLM call failed",
                          original_error=e) from e
```

### 7.3.1 AnthropicProvider

```python
# llm/anthropic_provider.py
import anthropic
from dataclasses import dataclass, field

@dataclass
class LLMUsage:
    input_tokens: int = 0
    output_tokens: int = 0

@dataclass
class LLMResponse:
    text: str
    stop_reason: str
    tool_calls: list[dict] = field(default_factory=list)
    usage: LLMUsage = field(default_factory=LLMUsage)

class AnthropicProvider:
    """Claude API 封装。单一职责：SDK 调用 → 统一 LLMResponse。"""
    def __init__(self, api_key: str, model: str = "claude-sonnet-4-6"):
        self.client = anthropic.Anthropic(api_key=api_key)
        self.model = model

    def create_message(
        self,
        system: str,
        messages: list[dict],
        tools: list[dict] | None = None,
        max_tokens: int = 4096,
    ) -> LLMResponse:
        kwargs = dict(
            model=self.model,
            system=system,
            messages=messages,
            max_tokens=max_tokens,
        )
        if tools:
            kwargs["tools"] = tools
        response = self.client.messages.create(**kwargs)

        tool_calls = []
        for block in response.content:
            if block.type == "tool_use":
                tool_calls.append({"id": block.id, "name": block.name, "input": block.input})

        text = "".join(block.text for block in response.content if block.type == "text")
        return LLMResponse(
            text=text,
            stop_reason=response.stop_reason,
            tool_calls=tool_calls,
            usage=LLMUsage(
                input_tokens=response.usage.input_tokens,
                output_tokens=response.usage.output_tokens,
            ),
        )
```

> **注意**：Claude API 的结构化输出通过 system prompt 中的 JSON 格式说明 + `tools` 的 `input_schema` 实现，**不使用** OpenAI 风格的 `response_format` 参数。`Task.output_schema` 应合并到 system prompt 中作为输出格式约束。

### 7.4 错误处理流程

| 类型 | 处理 | 恢复策略 |
|------|------|---------|
| LLM 调用失败（5xx/超时） | RetryPolicy 自动重试 | 继续当前 Stage |
| Guardrail 返回 action="drop" | 静默丢弃，继续下一 Stage | 自动 |
| Guardrail 返回 action="block" | 记录到 checkpoint，进入 error 状态 | 人工介入 |
| Guardrail 返回 action="retry" | 抛出 GuardrailRetrySignal → loop 重试 | 自动（限 3 次） |
| Stage 重试 > 3 次 | 抛 AEError(ErrorCode.STAGE_RETRY_EXCEEDED) | 人工介入 |
| CancellationToken.cancel() | 保存 checkpoint → raise LoopDrained | 可从 checkpoint 恢复 |
| Ctrl-C (SIGINT) | token.cancel() → 保存 checkpoint | 同上 |

```python
# engine/loop.py — run() 方法的重试+取消扩展（相对于 Section 二.1 基线的增量）
# 注意：完整 run() 见 Section 二.1（含 __init__/OutputDropped/runtime None 检查等）。
# 此处只展示 GuardrailRetrySignal 重试循环的集成方式。

# 在 Section 二.1 run() 的 while 循环内，处理 execute 的异常：
                try:
                    result = await self.runtime.execute(
                        self.current_task, self.checkpoint.state,
                        cancellation=self._cancellation,
                    )
                except OutputDropped:
                    continue  # 静默丢弃，下一轮
                except GuardrailRetrySignal as e:
                    retry_count += 1
                    if retry_count >= 3:
                        raise AEError(ErrorCode.STAGE_RETRY_EXCEEDED,
                                      f"Stage {self.current_task.name} 重试超过 3 次",
                                      original_error=e)
                    await asyncio.sleep(2 ** retry_count)
                    continue  # 跳过 after_tick，重试当前 Stage
                self.after_tick(result)
                retry_count = 0  # 成功后重置
```

> **注意**：`run()` 的完整版本在 Section 二.1。此处仅展示 `GuardrailRetrySignal` 与 `OutputDropped` 的异常处理集成方式。两者合并后即为最终 `run()` 实现。

```python
# tools/registry.py
class ToolRegistry:
    """字符串 → 工具实例映射。参考 CrewAI tools: list[BaseTool] 直接传实例模式。"""
    def __init__(self):
        self._tools: dict[str, BaseTool] = {}

    def register(self, tool: BaseTool) -> None:
        self._tools[tool.name] = tool

    def resolve(self, names: list[str]) -> list[BaseTool]:
        missing = [n for n in names if n not in self._tools]
        if missing:
            raise AEError(ErrorCode.TASK_NOT_FOUND,
                          f"Tools not found: {missing}")
        return [self._tools[n] for n in names]


# agents/parser.py
import json, re

def parse_agent_output(response_text: str, output_schema: dict | None = None) -> dict:
    """双层防御解析 LLM 输出。
    Layer 1: 完整 JSON parse（schema 模式优先）
    Layer 2: 正则提取 JSON 块（fallback，参考 CrewAI _JSON_PATTERN）
    """
    try:
        return json.loads(response_text)
    except json.JSONDecodeError:
        pass

    match = re.search(r'\{[^{}]*\{.*\}[^{}]*\}|\[.*\]|\{.*\}', response_text, re.DOTALL)
    if match:
        try:
            return json.loads(match.group(0))
        except json.JSONDecodeError:
            pass

    raise AEError(ErrorCode.INVALID_AGENT_OUTPUT,
                  f"Cannot parse LLM output: {response_text[:200]}...")
```

## 十、与旧设计的关键差异

| 维度 | 旧设计 | 新设计 | 理由 |
|------|--------|--------|------|
| Guardrail 模型 | `loop.py` 硬编码 + 二元 `ctx.blocked` | GuardrailChain + GuardrailResult(success, action) 四态 | CrewAI GuardrailResult + AutoGen DropMessage |
| Task 模型 | `description: str` | `Task(description, expected_output, context, tools, output_schema)` | CrewAI 富 Task + output_schema 约束 LLM 输出 |
| Stage 通信 | 模糊的 "state 传递" | 明确的 `input_channels` / `output_channels` | LangGraph channel 模型的简化版 |
| 反馈回路 | 说"回到 Developer"但没设计传什么 | `critic_feedback` channel → Developer 的 description_template 自动引用 | 解决了核心的功能缺失 |
| 多 Agent | 说"预留"无具体设计 | Send/PUSH 机制 + `_pending_sends` channel | LangGraph Send 是动态路由的唯一正确方式 |
| LLM 抽象 | AnthropicProvider/OpenAIProvider/LocalProvider 三层 | 只用 AnthropicProvider + output_schema | YAGNI：先做好一个 |
| 模板策略 | Jinja2 用于所有渲染 | init 用 Jinja2，loop 用 `{var}` str.format | 三个参考项目共识：简单场景不用模板引擎 |
| crew/ 目录 | 3 文件独立编排层 | 合并到 engine/graph.py + runtime/task.py | 减少间接层，编排是 graph 的职责 |
| 错误处理 | 裸字符串 + GateFailedError | ErrorCode Enum + AEError + RetryPolicy + CancellationToken | 参考 LangGraph + AutoGen |
| 错误恢复 | 3 行伪代码 | resume() + CancellationToken + GraphDrained | 对标 LangGraph interrupt_before/after + AutoGen cancel |

## 十一、已知限制与生产就绪待办

> 以下条目在 v1.0 设计中已识别，但因优先级排入后续 Phase 或 v2.0 实现。标记 `[Phase N]` 的项已纳入对应实施阶段。

### 可观测性 [Phase 4]

- 每个 Stage 开始/结束时输出结构化日志（stage name, elapsed, token_usage, success/fail）
- `--verbose` 标志输出 Agent-LLM 完整交互（prompt + response）
- 错误日志包含完整上下文（checkpoint_id, state snapshot, stack trace）
- 参考：LangGraph LangSmith 集成、AutoGen OpenTelemetry、CrewAI verbose callback

### Token 预算与成本控制 [Phase 4]

- 从 Anthropic API response 提取 `usage.input_tokens` / `usage.output_tokens`
- 单次 `dev-loop` 累计 token 计数
- `--max-tokens` CLI flag 设置总预算上限
- 完成时输出 token 消耗报告

### 并发执行保护 [Phase 2]

- SQLite checkpoint 文件加 `thread_id` 隔离（schema 已有该字段）
- 启动时检查是否已有同 project 的 loop 在运行（pidfile 或 advisory lock）
- 参考：LangGraph `checkpoint_ns` + `thread_id` 双重隔离

### 环境前置校验 [Phase 2]

- Git 仓库状态检查（uncommitted changes → 警告或 `--allow-dirty`）
- `.ae-answers.yml` 存在性检查（提示用户先运行 `ae init` 或 `ae dev-loop --detect`）
- Anthropic API key 有效性验证（发一个最小 API 调用确认连通）
- 磁盘空间检查（checkpoint 目录所在分区 > 100MB）

### Dry-run 模式 [Phase 4]

- `ae dev-loop --dry-run "需求"` — 只运行 Architect Stage，输出 plan 后退出
- 不修改文件、不执行 git 操作、不写入 checkpoint
- 参考：CrewAI `CrewPlanner`

### 工具失败策略 [Phase 3]

- 每个工具的 `execute()` 返回 `ToolResult(success, output, error)` 
- Agent 根据失败类型决定：继续 / 重试 / 回滚 / 报告用户
- `git_commit` 失败时回滚文件变更（`git checkout -- <files>`）

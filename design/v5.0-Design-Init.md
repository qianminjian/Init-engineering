## 1. 产品定位

Init Engineering 是**独立的 Agent Skill** — 为 Claude Code agent 提供项目环境初始化能力。不是 loop 的启动阶段，而是独立存在的初始化工具。

两种模式：
- **存量项目**：通过代码分析自动识别项目类型、依赖、配置
- **新项目**：向导式询问确认方向，生成定制化项目骨架

| 维度 | 描述 |
|------|------|
| 形态 | 5 阶段流水线 (detect / prompt / render / tasks / finalize) |
| 借鉴 | Copier (5 阶段 + !include) + Cookiecutter (progress bar + hooks) + Yeoman (composable features) + SST (auto-detect) |
| 8 项目类型 | app-service, library, cli-tool, skill, hook, mcp-server, spec-doc, monorepo |
| 4 语言 | TypeScript, Python, Go, Rust (32 个有效组合) |
| 存量项目 | 自动检测, 只增不改, 不修改 `src/` 任何已有文件 |

## 2. 5 阶段流水线 (Copier 借鉴)

```
┌────────────────────────────────────────────────────────────┐
│ InitWorker.execute() / Worker.run_copy()                    │
│                                                              │
│  Phase 1: detect                                            │
│    └─ ProjectDetector.detect(target_dir)                      │
│       └─ list_candidates() → FRAMEWORK_SIGNATURES 匹配       │
│       - 扫 package.json / pyproject.toml / Cargo.toml       │
│       - 扫 .claude/ / lefthook.yml / .github/               │
│       - 扫 src/ / lib/ / tests/ / docs/ / design/          │
│       - 输出 FRAMEWORK_SIGNATURES 8 类型匹配                 │
│       - 单匹配 → auto-detect; 0 匹配 → AskUserQuestion      │
│                                                              │
│  Phase 2: prompt                                            │
│    └─ InteractivePrompt.collect_answers(questions)           │
│       - 加载 ae-template.yml, 解析 Question 列表              │
│       - 6 层优先级: cli > interactive > previous >          │
│         defaults > builtins > external (AnswersMap)          │
│       - 渲染 Jinja2 when/validator                          │
│       - Ctrl-C → save_partial()                             │
│       - 支持 secret/multiselect/placeholder/qmark            │
│                                                              │
│  Phase 3: render (临时目录)                                  │
│    └─ TemplateRenderer.render_to(tmpdir)                    │
│       - 遍历多层 template_dirs (_shared, _features, type)    │
│       - .jinja 后缀(可配置): 双层渲染 (文件名 + 内容)       │
│       - SandboxedEnvironment (防 Jinja2 注入)               │
│       - 冲突处理: overwrite/skip/ask (Copier 模式)           │
│       - 渲染失败 → raise TemplateRenderError                 │
│       - preserve_symlinks: 保留 symlink 而非解析为内容       │
│                                                              │
│  Phase 4: tasks (临时目录执行)                               │
│    └─ TaskRunner.run(tasks_before, tasks_after)            │
│       - 渲染 when/cmd (Jinja2)                             │
│       - 设置 working_directory + extra_env                   │
│       - subprocess.run(cmd, list 模式无 shell 注入)          │
│       - returncode != 0 → raise TaskExecutionError          │
│                                                              │
│  Phase 5: finalize (原子移动)                                │
│    └─ shutil.copytree(tmpdir, dst_dir)                       │
│       - 写入 .ae-answers.yml                                 │
│       - 清理临时目录                                         │
│       - 失败回滚: 仅删 InitWorker 自己创建的目录             │
│                                                              │
│  ── 可选: run_update() ──                                   │
│    更新已有项目: 检测变更 → 执行 migration_tasks              │
│    支持 before_migration / after_migration 钩子              │
│                                                              │
└────────────────────────────────────────────────────────────┘
```

**与 Copier 的主要差异**：
- 去掉 Copier 的 `run_recopy`（重拷贝）、`run_update` 的 VCS 克隆相关逻辑
- 简化 context：去掉 `_copier_phase`、`_folder_name` 等内部变量
- 不做 Copier 的 `unsafe` 模板检查（Agent Skill 模式信任模板来源）

### 2.1 InitWorker 完整接口签名 (scaffold_phases.py)

```python
from dataclasses import dataclass, field
from pathlib import Path
from typing import Any

@dataclass
class InitResult:
    """execute() 返回结果."""
    dst_path: Path
    files: list[Path] = field(default_factory=list)
    answers: dict[str, Any] = field(default_factory=dict)
    project_type: str = ""


@dataclass
class InitWorker:
    """5 阶段流水线编排器 (参考 Copier Worker).

    用法:
        with InitWorker(dst_path=Path("my-project")) as worker:
            result = worker.execute()
    """
    # --- 构造参数 ---
    dst_path: Path
    project_type: str | None = None
    package_manager: str | None = None
    ci_platform: str | None = None
    test_runner: str | None = None
    use_typescript: bool | None = None
    use_lefthook: bool | None = None
    force: bool = False
    defaults: bool = False
    overwrite: bool = False
    quiet: bool = False
    pretend: bool = False
    skip_tasks: bool = False
    cleanup_on_error: bool = True
    incremental: bool = False

    # --- 内部状态 (init=False) ---
    _current_phase: str = field(init=False, default="")
    _template: "TemplateConfig" = field(init=False, default=None)
    _answers: "AnswersMap" = field(init=False, default_factory="AnswersMap")
    _cleanup_hooks: list[Callable[[], None]] = field(init=False, default_factory=list)
    _previous_answers: "AnswersMap | None" = field(init=False, default=None)
    _created_files: set[str] = field(init=False, default_factory=set)
    _mode: str = field(init=False, default="fresh")  # "fresh" | "incremental"

    # --- 上下文管理器 ---
    def __enter__(self) -> "InitWorker": ...
    def __exit__(self, exc_type, exc_val, exc_tb) -> bool: ...

    # --- 主入口 ---
    def execute(self) -> InitResult:
        """执行完整 5 阶段流水线.

        Returns:
            InitResult: 生成结果

        Raises:
            InitInterruptedError: Ctrl-C 中断
            ConfigFileError: 模板不存在或版本不兼容
            TargetDirectoryError: 目录非空且无 --force
            UnsatisfiedPrerequisiteError: 缺少 git/python
            TemplateRenderError: 渲染失败
            TaskExecutionError: 钩子执行失败
        """
        ...

    # --- 阶段方法 ---
    def _phase_detect(self) -> None:
        """Phase 1: 检测项目类型.

        - 检测目录状态 (fresh/incremental)
        - 自动检测或交互式询问项目类型
        - 前置条件检查 (git/python)
        """
        ...

    def _phase_prompt(self) -> None:
        """Phase 2: 交互式问答.

        - 加载 ae-template.yml
        - CLI overrides 注入 AnswersMap
        - 交互式提问 (支持 secret/multiselect/placeholder)
        - Ctrl-C → save_partial()
        """
        ...

    def _phase_render(self, tmpdir: Path) -> list[Path]:
        """Phase 3: 渲染到临时目录.

        Args:
            tmpdir: 临时目录路径

        Returns:
            生成的文件路径列表

        Raises:
            TemplateRenderError: 渲染失败
        """
        ...

    def _phase_tasks(self, tmpdir: Path) -> None:
        """Phase 4: 执行钩子任务.

        Args:
            tmpdir: 临时目录路径

        Raises:
            TaskExecutionError: 任务执行失败
        """
        ...

    def _phase_finalize(self, tmpdir: Path, generated: list[Path]) -> bool:
        """Phase 5: 原子移动 + 写入配置.

        Args:
            tmpdir: 临时目录路径
            generated: 生成的文件列表

        Returns:
            did_create_dst: 是否本次创建了目标目录

        Raises:
            OSError: 文件操作失败
        """
        ...

    # --- 辅助方法 ---
    def _check_template_version(self) -> None:
        """检查 ae 版本是否满足模板要求的最小版本.

        Raises:
            ConfigFileError: 版本不满足
        """
        ...

    def _check_prerequisites(self) -> None:
        """检查必需工具 (git/python).

        Raises:
            UnsatisfiedPrerequisiteError: 工具缺失
        """
        ...

    def _cleanup(self) -> None:
        """执行所有注册的清理回调."""
        ...
```

### 2.2 InitWorker 内部状态转换

```
execute() 状态机:
┌─────────────────────────────────────────────────────┐
│ _mode: "fresh" | "incremental"                      │
│                                                     │
│ execute()                                           │
│   ↓                                                 │
│ _phase_detect()                                    │
│   ├─ dst_path 不存在 → _mode = "fresh"             │
│   ├─ dst_path 空目录 → _mode = "fresh"             │
│   └─ dst_path 非空 + incremental → _mode = "incremental" │
│   └─ dst_path 非空 + 无 incremental → TargetDirectoryError │
│   ↓                                                 │
│ _phase_prompt() → _answers                          │
│   ↓                                                 │
│ tmpdir = mkdtemp()                                 │
│ _cleanup_hooks.append(rmtree(tmpdir))              │
│   ↓                                                 │
│ _phase_render(tmpdir) → generated                   │
│   ↓                                                 │
│ _phase_tasks(tmpdir)                               │
│   ↓                                                 │
│ _phase_finalize(tmpdir, generated)                 │
│   ├─ _mode == "incremental" → merge_incremental   │
│   └─ _mode == "fresh" → copytree                  │
│   ↓                                                 │
│ return InitResult                                   │
│   ↓                                                 │
│ _cleanup() → rmtree(tmpdir)                        │
└─────────────────────────────────────────────────────┘
```

### 2.3 InitWorker 与 Copier Worker 对照

| 方法 | Copier Worker | 我们的 InitWorker | 差异 |
|------|--------------|-------------------|------|
| `execute()` | `run_copy()` | `execute()` | 相同 |
| `_phase_detect()` | 无对应 | 项目检测 | AE 特有 |
| `_phase_prompt()` | `_ask()` | `_phase_prompt()` | 简化 |
| `_phase_render()` | `_render_template()` | `_phase_render()` | 相同 |
| `_phase_tasks()` | `_execute_tasks()` | `_phase_tasks()` | 相同 |
| `_phase_finalize()` | 无单独方法 | `_phase_finalize()` | AE 特有 |
| `run_update()` | 支持 | 不支持 | P1 缺失 |
| `run_recopy()` | 支持 | 不支持 | P2 |

## 4. 自动检测 (存量项目, SST 借鉴)

```python
# ProjectDetector: 扫已知文件签名, 推断项目类型
FRAMEWORK_SIGNATURES: list[tuple[str, list[str]]] = [
    ("monorepo", ["pnpm-workspace.yaml", "lerna.json", "turbo.json", "nx.json"]),
    ("skill", [".claude/skills/"]),
    ("hook", [".claude/hooks/"]),
    ("spec-doc", ["design/BEACON.md", "design/*.md"]),
    ("mcp-server", ["package.json"]),  # + 二次校验 @modelcontextprotocol/sdk
    ("cli-tool", ["package.json"]),     # + 二次校验 "bin" 字段
    ("library", ["pyproject.toml", "setup.py", "Cargo.toml", "go.mod"]),
    ("app-service", ["package.json"]),
]

class ProjectDetector:
    def detect(self) -> str | None:
        candidates = self.list_candidates()
        return candidates[0] if len(candidates) == 1 else None

    def list_candidates(self) -> list[str]:
        matches = []
        for ptype, sigs in FRAMEWORK_SIGNATURES:
            if any(self._match(s) for s in sigs):
                if self._verify_advanced(ptype):
                    matches.append(ptype)
        return matches
```

**init 行为 (存量项目)**:
- list_candidates = [] → 空目录, AskUserQuestion 1 问
- list_candidates = [N] (单匹配) → auto-detect
- list_candidates = [N1, N2, ...] (多匹配) → AskUserQuestion 1 问 (选置信度最高的)

## 5. AnswersMap (6 层 ChainMap, Copier 借鉴)

```python
@dataclass
class AnswersMap:
    """6 层优先级 ChainMap (简化自 Copier 的 8 层).

    优先级 (高 → 低):
      1. cli_overrides  — --flag 参数
      2. interactive    — 交互式问答输入
      3. previous       — --from-answers 加载历史
      4. defaults       — ae-template.yml 中 default
      5. builtins       — 内置变量 (_ae_version 等)
      6. external       — 外部数据懒加载 (YAML/JSON)
    """
    cli_overrides: dict = field(default_factory=dict)
    interactive: dict = field(default_factory=dict)
    previous: dict = field(default_factory=dict)
    defaults: dict = field(default_factory=dict)
    builtins: dict = field(default_factory=lambda: BUILTIN_VARS.copy())
    external: dict[str, str] = field(default_factory=dict)  # lazy-load YAML/JSON
    hidden: set = field(default_factory=set)  # 不写入 .ae-answers.yml

    def get(self, key: str) -> Any:
        for layer in [self.cli_overrides, self.interactive, self.previous,
                      self.defaults, self.builtins]:
            val = layer.get(key)
            if val is not None:
                return val
        if key in self.external:
            return self._load_external(key)
        raise KeyError(key)

    def save_partial(self) -> Path:
        """Ctrl-C 时保存部分答案, 下次 --from-answers 恢复."""
        path = Path.home() / ".ae-partial-answers.yml"
        yaml.dump({"_meta": {"saved_at": datetime.now().isoformat(),
                              "partial": True},
                   **self.interactive}, path)
        return path
```

### 5.2 外部数据懒加载 (Copier _external_data 借鉴)

```python
class _LazyExternalDict:
    """懒加载外部 YAML/JSON 数据文件.

    来源: Copier _external_data + LazyDict 模式.
    v2.5 P1-S3: external_data 路径沙箱校验.
    """
    def __init__(self, external_map: dict[str, str],
                 sandbox_roots: list[Path] | None = None):
        self._external_map = external_map  # {var_name: file_path}
        self._sandbox_roots = sandbox_roots or []
        self._cache: dict[str, Any] = {}

    def __getitem__(self, key: str) -> Any:
        # sandbox 检查: 防止模板读取 /etc/passwd 等
        if key not in self._cache:
            file_path = Path(self._external_map[key])
            if self._sandbox_roots and not _is_path_under_any_root(
                file_path, self._sandbox_roots
            ):
                raise ValueError(
                    f"external_data path '{file_path}' not under sandbox roots"
                )
            # YAML (.yml/.yaml) 或 JSON (.json) 加载
            if file_path.suffix in (".yml", ".yaml"):
                data = yaml.safe_load(file_path.read_text())
            elif file_path.suffix == ".json":
                import json
                data = json.loads(file_path.read_text())
            else:
                data = yaml.safe_load(file_path.read_text())
            self._cache[key] = data
        return self._cache[key]
```

## 6. ae-template.yml (YAML 配置, Copier 借鉴)

```yaml
# templates/app-service/ae-template.yml
_min_ae_version: "5.0.0"
_envops:
  autoescape: false
  keep_trailing_newline: true
_exclude: []
_skip_if_exists: []
_no_render: ["**/*.png", "**/*.ico"]

    when: "{{ not project_exists('.git') }}"

  type: str
  help: 项目名称 (kebab-case)
  validator: "{{ 'project_name' in locals() and re.match(r'^[a-z0-9-]+$', value) or '必须小写字母+数字+连字符' }}"

project_description:
  type: str
  help: 项目简短描述
  default: ""
```

## 7. TemplateRenderer (Jinja2 双层渲染, Copier 借鉴)

### 7.1 完整接口签名

```python
from collections.abc import Callable
from pathlib import Path
import jinja2
from jinja2 import StrictUndefined
from jinja2.sandbox import SandboxedEnvironment
import pathspec
import shutil

class TemplateRenderer:
    """遍历多层模板目录, 双层渲染文件名 + 内容.

    借鉴:
      - Copier _main.py:769-900 (文件渲染 + 冲突处理)
      - Cookiecutter generate.py:175-260 (binary 检测 + 换行符保持)
    """

    TEMPLATE_SUFFIX: str = ".jinja"   # 硬编码模板后缀，不可配置
    env: SandboxedEnvironment

    def __init__(
        self,
        template_dirs: list[Path],
        context: dict[str, object],
        exclude: list[str] | None = None,
        skip_if_exists: list[str] | None = None,
        no_render: list[str] | None = None,
        overwrite: bool = False,
        conflict_handler: Callable[[str], bool] | None = None,
        match_exclude: Callable[[Path], bool] | None = None,
    ) -> None:
        """初始化渲染器.

        Args:
            template_dirs: 模板搜索路径列表(优先级从低到高)
            context: Jinja2 渲染上下文
            exclude: gitignore 风格排除模式
            skip_if_exists: 已存在时跳过的模式
            no_render: 原样复制的文件模式
            overwrite: 是否覆盖已存在文件
            conflict_handler: 冲突回调，返回 True=覆盖，False=跳过
            match_exclude: 额外排除回调（来自 Copier match_exclude）
        """
        ...

    def render_to(self, dst_dir: Path) -> list[Path]:
        """渲染所有模板文件到目标目录.

        Returns:
            生成的目标文件路径列表

        Raises:
            TemplateRenderError: Jinja2 渲染失败或路径穿越
            OSError: 文件操作失败
        """
        ...

    # --- 内部方法 ---

    def _render_path(self, path_str: str) -> str:
        """渲染路径模板字符串.

        Args:
            path_str: 相对路径模板字符串

        Returns:
            渲染后的路径字符串(空字符串表示条件跳过)

        Raises:
            TemplateRenderError: Jinja2 渲染失败
        """
        ...

    def _render(self, content: str) -> str:
        """渲染文件内容模板.

        Args:
            content: 模板文件内容

        Returns:
            渲染后的内容

        Raises:
            TemplateRenderError: Jinja2 渲染失败
        """
        ...

    def _path_matcher(self, patterns: list[str]) -> Callable[[str], bool]:
        """构建路径匹配器.

        Args:
            patterns: gitignore 风格模式列表

        Returns:
            匹配函数 (path: str) -> bool
        """
        ...

    def _is_excluded(self, file_path: Path, src_dir: Path) -> bool:
        """检查文件是否被排除.

        两层检查:
        1. self.exclude 模式匹配
        2. self.match_exclude 回调(可选)
        """
        ...

    def _is_no_render(self, rel_path: str) -> bool:
        """检查文件是否应原样复制不渲染."""
        ...

    def _should_overwrite(self, rel_path: str) -> bool:
        """判断是否应覆盖已存在的文件.

        优先级:
        1. self.overwrite = True → 覆盖
        2. self.skip_if_exists 匹配 → 不覆盖
        3. self.conflict_handler 回调 → 使用回调结果
        4. 默认 → 不覆盖
        """
        ...

    # 注意: 路径穿越检查内联在 render_to() 中（非独立方法）
    # render_to() 内使用 os.path.realpath 双侧归一化（macOS symlink 安全）
    # 文件不存在时回退到 str(dst_file.resolve())

    @staticmethod
    def _detect_newline(file_path: Path) -> str | None:
        """检测文件换行符风格.

        Returns:
            '\n' / '\r\n' / None(无法检测)
        """
        ...
```

### 7.2 冲突处理决策流程

```
文件冲突检测点 (_phase_render 内):
│
├── dst_file 不存在
│   └── → 直接渲染/复制
│
├── dst_file 存在
│   ├── self.overwrite = True
│   │   └── → 覆盖
│   │
│   ├── skip_if_exists 模式匹配
│   │   └── → 跳过 (不覆盖)
│   │
│   ├── self.conflict_handler 回调存在
│   │   └── → 调用 conflict_handler(rel_path)
│   │       ├── 返回 True → 覆盖
│   │       └── 返回 False → 跳过
│   │
│   └── 无配置 → 跳过 (不覆盖)
│
└── 渲染时异常
    └── → raise TemplateRenderError
```

### 7.3 渲染流程时序

```
TemplateRenderer.render_to(dst_dir)
│
├── for src_dir in template_dirs:
│   └── for src_file in src_dir.rglob("*"):
│       ├── _is_excluded() → 跳过
│       ├── _render_path(rel_path) → 渲染文件名
│       │   └── 渲染结果为空 → 跳过(条件渲染)
│       ├── is_template = endswith(templates_suffix)
│       ├── _is_safe_path() → 路径穿越检查
│       │   └── 不安全 → raise TemplateRenderError
│       ├── dst_file.exists()?
│       │   └── 存在 + not _should_overwrite() → 跳过
│       ├── _is_no_render() → 原样复制
│       ├── src_file.is_symlink() and preserve_symlinks?
│       │   └── 是 → symlink_to(target)
│       ├── is_template?
│       │   ├── True → _render(content) + write_text(newline)
│       │   └── False → is_binary()?
│       │       ├── True → copy2 (二进制)
│       │       └── False → copy2 + newline
│       └── generated[rel_path] = dst_file
│
└── return list[Path]
```

```python
def _phase_finalize(self, tmpdir, generated):
    """原子移动 + 不覆盖已有."""
    if self.incremental:
        for f in generated:
            dst = self.dst_path / f
            if dst.exists():
```

## 10. .ae-answers.yml 配置契约

> **v5.0 起 Loop 已裁剪，本章节保留 ProjectEnvironment 作为通用配置读取机制。**

### 10.1 设计原则

```
ae init 创建了项目 → .ae-answers.yml 是权威来源
项目不是 ae init 创建的 → 运行时自检测生成 .ae-answers.yml
代码与 .ae-answers.yml 不一致 → 以代码为准，分层处理
```

### 10.2 信息分类

| 类型 | 字段示例 | 检测方式 | 不一致时 |
|------|---------|---------|---------|
| **可客观判定** | `package_manager` | 检查 lock 文件 | 以代码为准，静默更新 |
| | `use_typescript` | `tsconfig.json` 是否存在 | 同上 |
| | `ci_platform` | `.github/workflows/` / `.gitlab-ci.yml` | 同上 |
| | `test_runner` | 测试框架配置文件 | 同上 |
| | `has_lefthook` | `lefthook.yml` 是否存在 | 同上 |
| **不可客观判定** | `project_name` | — | 代码无法判断，不改 |
| | `project_description` | — | 同上 |
| | `project_type` (8 种) | 启发式签名匹配，可能模糊 | 有冲突时提示用户，不改 |

### 10.3 ProjectEnvironment (配置读取契约)

```python
class ProjectEnvironment:
    """.ae-answers.yml 配置读取 + 自检测."""
    def __init__(self, project_root: Path):
        self.answers_path = project_root / ".ae-answers.yml"
        self.answers = self._load_or_detect(project_root)

    def _load_or_detect(self, root: Path) -> dict:
        if self.answers_path.exists():
            data = yaml.safe_load(self.answers_path.read_text())
            return self._merge_with_detection(root, data)
        return self._detect_from_scratch(root)

    def _merge_with_detection(self, root, recorded):
        """可判定项不一致 → 以代码为准, 静默更新."""
        detected = self._detect_from_scratch(root)
        for key in ("package_manager", "use_typescript", "ci_platform",
                     "test_runner", "has_lefthook"):
            if key in detected and key in recorded \
                    and detected[key] != recorded[key]:
                recorded[key] = detected[key]
        for key in ("project_name", "project_description", "project_type"):
            if key in detected and key in recorded \
                    and detected[key] != recorded[key]:
                print(f"提示: 记录的 {key}={recorded[key]}, 当前代码像 {detected[key]}")
        return recorded
```

| `ae init --analyze` (只分析) | ❌ | ❌ | ❌ |

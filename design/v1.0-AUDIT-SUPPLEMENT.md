## 新增发现（10 个优化点）

### P0-18：Gate 模型应升级为 Guardrail

**来源**：CrewAI `GuardrailResult(success, result, error)` + AutoGen `DropMessage`

**现状问题**：Gate 只有 `ctx.blocked = True/False` 二元语义。无法表达"通过但需要修正""丢弃此消息但不中止整个循环"等语义。

**优化方案**：

```python
# gates/guardrail.py
@dataclass
class GuardrailResult:
    success: bool
    result: Any | None = None        # 通过时可能转换的输出
    error: str = ""                  # 失败原因
    action: str = "block"            # "pass" | "block" | "drop" | "retry"

class DropOutput:
    """AutoGen 风格 sentinel：静默丢弃此输出，不中止循环"""
    pass
```

GuardrailChain 替代 InterventionChain——每个 Guardrail 返回 GuardrailResult，链根据 action 决定下一步。

**影响范围**：`gates/chain.py` → 重命名 `gates/guardrail.py`，`gates/builtin.py` 中 5 个 Gate 升级为 Guardrail

### P0-19：Agent 输出格式应使用 output_schema 约束

**来源**：CrewAI `output_json` / `output_pydantic`

**现状问题**：设计用 `_parse_output()` 正则提取 JSON——脆弱，LLM 输出格式不稳定时解析失败。

**优化方案**：

```python
# runtime/task.py
@dataclass
class Task:
    description: str
    expected_output: str
    context: dict
    tools: list[str]
    output_schema: dict | None = None   # JSON Schema，传给 Claude API 的 response_format
```

Claude API 支持 `tool_choice` 和 structured output——传入 schema 后 LLM 直接返回符合 schema 的 JSON，不再需要正则提取。

Agent 的 `_parse_output()` 退化为 schema validation + fallback regex（双层防御）。

**影响范围**：`runtime/task.py`，`agents/base.py`，`agents/parser.py`

### P0-20：ErrorCode 体系缺失

**来源**：LangGraph `ErrorCode(Enum)` + `create_error_message()`

**现状问题**：设计中错误都是裸字符串，没有结构化标识。排查问题时只能靠 grep 错误消息。

**优化方案**：

```python
# errors.py
from enum import Enum

class ErrorCode(Enum):
    CHECKPOINT_SAVE_FAILED = "CHECKPOINT_SAVE_FAILED"
    LLM_TIMEOUT = "LLM_TIMEOUT"
    LLM_MAX_RETRIES = "LLM_MAX_RETRIES"
    GUARDRAIL_BLOCKED = "GUARDRAIL_BLOCKED"
    STAGE_RETRY_EXCEEDED = "STAGE_RETRY_EXCEEDED"
    INVALID_AGENT_OUTPUT = "INVALID_AGENT_OUTPUT"
    GRAPH_RECURSION_LIMIT = "GRAPH_RECURSION_LIMIT"
    TASK_NOT_FOUND = "TASK_NOT_FOUND"

class AEError(Exception):
    def __init__(self, code: ErrorCode, message: str, original_error: Exception | None = None):
        self.code = code
        self.message = message
        self.original_error = original_error
        super().__init__(f"[{code.value}] {message}")
```

**影响范围**：新增 `errors.py`，所有 raise 语句改用 `AEError`

### P1-21：RetryPolicy 应独立为可配置对象

**来源**：LangGraph `RetryPolicy` + `run_with_retry()`

**现状问题**（初版 P1-#9）：设计说"重试 2 次"但 hardcoded 在 execute() 中，不可配置。

**优化方案**：

```python
# engine/retry.py
@dataclass
class RetryPolicy:
    max_attempts: int = 3
    initial_delay: float = 1.0
    backoff_factor: float = 2.0
    max_delay: float = 60.0
    retry_on: tuple[type[Exception], ...] = (Exception,)
    timeout: float | None = None   # None = 无超时

async def run_with_retry(fn, policy: RetryPolicy) -> Any:
    """LangGraph 风格的集中式重试循环"""
    ...
```

每个 Stage 可独立配置 `retry_policy`。

**影响范围**：新增 `engine/retry.py`，`agents/base.py` 和 `engine/loop.py` 使用

### P1-22：缺少 CancellationToken

**来源**：AutoGen `CancellationToken`

**现状问题**：设计中 LLM 调用超时有 `asyncio.wait_for`，但没有主动取消机制。用户 Ctrl-C 后 checkpoint 保存逻辑没有保护。

**优化方案**：

```python
# engine/cancellation.py
class CancellationToken:
    def __init__(self):
        self._cancelled = False

    def cancel(self) -> None:
        self._cancelled = True

    @property
    def is_cancelled(self) -> bool:
        return self._cancelled

    def check(self) -> None:
        if self._cancelled:
            raise TaskCancelledError()
```

LoopEngine 持有 token，Agent 每次执行前 `token.check()`。Ctrl-C 触发 `token.cancel()` → 当前 Agent 优雅退出 → 保存 checkpoint。

**影响范围**：新增 `engine/cancellation.py`，`LoopEngine.run()` 和 `BaseAgent.execute()` 持有 token 引用

### P1-23：Template 策略应分类

**来源**：CrewAI 用 `{var}` 字符串插值，LangGraph 用 Python f-string，AutoGen 用 dict 替换

**现状问题**：Task 描述渲染用 Jinja2——对简单场景是过度工程。三个参考项目没有一个用 Jinja2 做 Agent prompt 渲染。

**优化方案**：

| 使用场景 | 渲染方式 | 理由 |
|---------|---------|------|
| init 模板文件渲染（文件名 + 内容，条件分支） | **Jinja2** | 需要条件/循环/过滤器，Jinja2 不可替代 |
| Task 描述渲染（简单变量替换） | **Python `str.format()`** | 只替换 `{plan}` `{feedback}` 等变量，不需要模板引擎 |
| Agent system_prompt（静态文本） | **不做渲染** | 静态字符串 + 通过 messages context 注入动态信息 |

Stage 的 `render_description()` 改为 `state.get_channels(names).__str_format__()` 模式。

**影响范围**：`engine/graph.py:Stage.render_description()`，移除 Jinja2 依赖（loop 子系统不再依赖 jinja2）

### P1-24：缺少优雅关闭机制

**来源**：LangGraph `GraphDrained` + AutoGen `QueueShutDown`

**现状问题**：设计中有 `interrupt` 状态但缺少 SIGTERM/Ctrl-C 的优雅处理——中断时 checkpoint 可能未保存。

**优化方案**：

```python
# engine/loop.py
class LoopEngine:
    async def run(self, ...) -> LoopResult:
        try:
            while True:
                self._cancellation.check()
                if not self.tick(max_steps):
                    break
                result = await self._execute_current_stage()
                self.after_tick(result)
        except TaskCancelledError:
            # 优雅关闭：保存 checkpoint，设置 drain 状态
            self.checkpoint.status = "drained"
            self.checkpoint.save()
            raise LoopDrained(self.checkpoint.id)
```

**影响范围**：`engine/loop.py:run()`

### P2-25：Tool 应类型化

**来源**：AutoGen `BaseTool[ArgsT, ReturnT]` Generic

**现状问题**：设计中的 `BaseTool.execute(**kwargs) -> ToolResult` 无类型约束。调用方不知道某个工具接受什么参数、返回什么。

**优化方案**：

```python
# tools/base.py
from typing import Generic, TypeVar

ArgsT = TypeVar("ArgsT", bound=BaseModel)
ReturnT = TypeVar("ReturnT")

class BaseTool(ABC, Generic[ArgsT, ReturnT]):
    name: str
    description: str
    args_schema: type[ArgsT]

    @abstractmethod
    async def execute(self, args: ArgsT) -> ReturnT: ...

    def to_schema(self) -> dict:
        """转换为 Claude API tool schema"""
        ...
```

**影响范围**：`tools/base.py`，4 个工具文件

### P2-26：Agent 注册应加类型验证

**来源**：AutoGen `register_factory` 类型检查模式

**现状问题**（初版 P1-#5 已提但未给具体实现）：设计中 `register(agent_type, factory)` 没有检查 factory 是否返回正确的 Agent 类型。

**优化方案**：

```python
# runtime/runtime.py
def register(self, agent_type: str, factory: Callable[[], BaseAgent], 
             expected_class: type[BaseAgent] | None = None) -> None:
    if expected_class is not None:
        instance = factory()
        if not isinstance(instance, expected_class):
            raise TypeError(
                f"Agent factory for '{agent_type}' returned {type(instance).__name__}, "
                f"expected {expected_class.__name__}"
            )
        # 即时销毁测试实例，正式使用时再创建
        del instance
    self._agent_factories[agent_type] = factory
```

**影响范围**：`runtime/runtime.py:register()`

### P2-27：输出解析应为双层防御

**来源**：综合 CrewAI Pydantic validation + LangGraph error handling

**现状问题**：`extract_json_from_response()` 只有一层正则——LLM 输出格式不符合预期时直接 ValueError。

**优化方案**：

```python
# agents/parser.py
def parse_agent_output(response_text: str, output_schema: dict | None = None) -> dict:
    """双层防御：
    1. 如设置了 output_schema → Claude API 已返回结构化 JSON → 直接 parse
    2. Fallback：正则提取 JSON 块（参考 CrewAI _JSON_PATTERN）
    3. 最后防线：尝试首尾截取
    """
    # Layer 1: 完整 JSON
    try:
        return json.loads(response_text)
    except json.JSONDecodeError:
        pass

    # Layer 2: JSON 块提取
    match = re.search(r'\{[^{}]*\{.*\}[^{}]*\}|\[.*\]|{.*}', response_text, re.DOTALL)
    if match:
        try:
            return json.loads(match.group(0))
        except json.JSONDecodeError:
            pass

    # Layer 3: 无法解析
    raise AEError(ErrorCode.INVALID_AGENT_OUTPUT, 
                  f"Cannot parse LLM output: {response_text[:200]}...")
```

**影响范围**：`agents/parser.py`

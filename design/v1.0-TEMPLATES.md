## 模板资产完整定义

### 17.1 项目类型 × 语言矩阵

| 项目类型 | 适用语言/生态 | 包管理器 | 测试框架 | Linter | Formatter |
|---------|-------------|---------|---------|--------|-----------|
| **app-service** | TypeScript/Node | npm/pnpm/yarn/bun | vitest/jest | eslint | prettier |
| | Python | uv/pip/poetry | pytest | ruff | ruff |
| | Go | go mod | go test | golangci-lint | gofmt/goimports |
| **library** | TypeScript/Node | 同上 | 同上 | 同上 | 同上 |
| | Python | 同上 | 同上 | 同上 | 同上 |
| | Go | go mod | go test | golangci-lint | gofmt |
| | Rust | cargo | cargo test | clippy | rustfmt |
| **cli-tool** | TypeScript/Node | 同上 | 同上 | 同上 | 同上 |
| | Python | 同上 | 同上 | 同上 | 同上 |
| | Go | go mod | go test | golangci-lint | gofmt |
| | Rust | cargo | cargo test | clippy | rustfmt |
| **skill** | TypeScript/Node | pnpm | vitest | eslint | prettier |
| **hook** | Bash | — | bats/shellcheck | shellcheck | shfmt |
| **mcp-server** | TypeScript/Node | pnpm | vitest | eslint | prettier |
| | Python | uv | pytest | ruff | ruff |
| **spec-doc** | 无（纯文档） | — | — | markdownlint | prettier |
| **monorepo** | TypeScript/Node（多子包） | pnpm workspace | vitest/jest | eslint | prettier |

### 17.3 8 种项目类型的 ae-template.yml

#### 17.3.1 app-service/ae-template.yml

```yaml
_min_ae_version: "1.0.0"
_templates_suffix: ".jinja"
_exclude: ["ae-template.yml", "~*", "*.py[co]", "__pycache__", ".git", ".DS_Store", ".svn"]

_message_before: "正在生成 {{ project_name }} 应用服务..."
_message_after: "✓ 应用服务已就绪。运行 cd {{ _dst_basename }} && {{ package_manager }} dev 启动。"

# ─── 基础信息 ───
project_name:
  type: str
  help: 项目名称
  default: "{{ _dst_basename }}"
  validator: >-
    {% if not (project_name | regex_search('^[a-z][a-z0-9\\-]+$')) %}
    项目名称为小写字母+数字+连字符
    {% endif %}

project_description:
  type: str
  help: 项目描述（一句话）
  default: ""

# ─── 语言选择 ───
language:
  type: str
  help: 语言/生态
  choices:
    TypeScript/Node: typescript
    Python: python
    Go: go
  default: typescript

# ─── TypeScript 专属 ───
package_manager:
  type: str
  help: 包管理器
  choices: [npm, pnpm, yarn, bun]
  default: pnpm
  when: "{{ language == 'typescript' }}"

use_typescript:
  type: bool
  help: 使用 TypeScript？
  default: yes
  when: "{{ language == 'typescript' }}"

ts_test_runner:
  type: str
  help: 测试框架
  choices: [vitest, jest]
  default: vitest
  when: "{{ language == 'typescript' and use_typescript }}"

# ─── Python 专属 ───
py_package_manager:
  type: str
  help: Python 包管理器
  choices:
    uv: uv
    pip: pip
    poetry: poetry
  default: uv
  when: "{{ language == 'python' }}"

py_python_version:
  type: str
  help: Python 版本
  choices: ["3.12", "3.13"]
  default: "3.12"
  when: "{{ language == 'python' }}"

# ─── Go 专属 ───
go_module_path:
  type: str
  help: Go module 路径（如 github.com/user/repo）
  default: "github.com/{{ project_name }}/{{ project_name }}"
  when: "{{ language == 'go' }}"

# ─── 通用基础设施 ───
use_lefthook:
  type: bool
  help: 安装 Lefthook Git hooks？
  default: yes

ci_platform:
  type: str
  help: CI 平台
  choices:
    无: none
    GitHub Actions: github
    GitLab CI: gitlab
  default: github

use_docker:
  type: bool
  help: 包含 Dockerfile？
  default: no

license:
  type: str
  help: 开源许可证
  choices: [MIT, "Apache-2.0", "GPL-3.0"]
  default: MIT

# ─── 钩子 ───
_tasks:
  - command: git init -b main
  - command: git add -A
  - command: git commit -m "chore(init): scaffolded by ae init"
    when: "{{ language == 'typescript' }}"
  - command: "{{ package_manager }} install"
    when: "{{ language == 'typescript' }}"
  - command: "{{ py_package_manager }} install"
    when: "{{ language == 'python' and py_package_manager == 'poetry' }}"
  - command: uv sync
    when: "{{ language == 'python' and py_package_manager == 'uv' }}"
  - command: pip install -e ".[dev]"
    when: "{{ language == 'python' and py_package_manager == 'pip' }}"
  - command: go mod tidy
    when: "{{ language == 'go' }}"
  - command: lefthook install
    when: "{{ use_lefthook }}"
  - command: git add -A && git commit -m "chore: install deps and hooks" --allow-empty
```

#### 17.3.4 skill/ae-template.yml

#### 17.3.5 hook/ae-template.yml

Claude Code Hook 模板。仅 Bash 生态。

```yaml
_min_ae_version: "1.0.0"
_templates_suffix: ".jinja"

hook_name:
  type: str
  help: Hook 名称（如 block-secrets-write）
  default: "{{ _dst_basename }}"

hook_type:
  type: str
  help: Hook 触发时机
  choices:
    PreToolUse: PreToolUse
    PostToolUse: PostToolUse
  default: PreToolUse

hook_description:
  type: str
  help: Hook 功能描述（一句话）
  default: ""

hook_tool_pattern:
  type: str
  help: 拦截的工具名（如 Write,Edit,Bash，* 表示所有）
  default: "*"
  when: "{{ hook_type == 'PreToolUse' }}"

hook_block_pattern:
  type: str
  help: 拦截的正则模式（如 rm -rf|password）
  default: ""
  when: "{{ hook_type == 'PreToolUse' }}"

_tasks:
  - command: git init -b main
  - command: git add -A
  - command: git commit -m "chore(init): scaffolded hook by ae init"
  - command: chmod +x "{{ hook_name }}.sh"
```

**hook 类型专属模板文件**：


### 17.8 TemplateRenderer 的特征目录选择逻辑

对应 `init/scaffold.py` 中 `_phase_render()` 的 template_dirs 构建：

```python
def _build_template_dirs(self, context: dict) -> list[Path]:
    """根据 AnswersMap 选择语言特征 + 基础设施特征。"""
    dirs = [TEMPLATES_ROOT / "_shared"]

    language = context.get("language", "typescript")

    # 语言特征
    lang_feature_map = {
        "typescript": ["typescript"],
        "python": ["python"],
        "go": ["go"],
        "rust": ["rust"],
        "bash": ["bash"],
    }
    for feat in lang_feature_map.get(language, []):
        dirs.append(TEMPLATES_ROOT / "_features" / feat)

    # 基础设施特征
    if context.get("use_lefthook"):
        dirs.append(TEMPLATES_ROOT / "_features" / "lefthook")

    ci = context.get("ci_platform", "none")
    if ci == "github":
        dirs.append(TEMPLATES_ROOT / "_features" / "github-actions")
    elif ci == "gitlab":
        dirs.append(TEMPLATES_ROOT / "_features" / "gitlab-ci")

    if context.get("use_docker"):
        dirs.append(TEMPLATES_ROOT / "_features" / "docker")

    if context.get("project_type") == "monorepo":
        dirs.append(TEMPLATES_ROOT / "_features" / "monorepo")

    # 项目类型模板（最高优先级）
    dirs.append(self._template.template_dir)

    return dirs
```
